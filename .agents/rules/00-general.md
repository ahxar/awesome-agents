# General Rules

## Priority

Follow these rules in order:

1. Security and data safety.
2. Correctness and reliability.
3. Maintainability and readability.
4. Performance.
5. Developer convenience.

## Behavior

- Make focused, minimal changes that solve the requested task.
- Prefer existing project patterns over introducing new styles.
- Do not remove or revert unrelated changes.
- State assumptions clearly when requirements are ambiguous.

## Safety

- Never commit secrets, tokens, or credentials.
- Avoid destructive operations unless explicitly requested.
- Call out risky changes before applying them.
